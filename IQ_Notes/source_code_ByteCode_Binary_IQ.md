# Source Code vs Byte Code vs Binary Code

## Example File: `01-hello-world.js`

```javascript
console.log('Hello world')
```

## Comparison Table

| Aspect | **Source Code** | **Byte Code** | **Binary Code (Machine Code)** |
|--------|----------------|---------------|--------------------------------|
| **What is it?** | Human-readable code written by a programmer in a high-level language | Intermediate representation between source code and machine code, executed by a **Virtual Machine (VM)** | Raw instructions in **binary (0s and 1s)** that the **CPU** executes directly |
| **Readability** | Readable by humans | Somewhat readable by experts (opcodes + operands) | Not readable by humans without a disassembler |
| **Example** (for `01-hello-world.js`) | `console.log('Hello world')` | V8 bytecode generated from parsing the above source: | Machine code executed by the CPU for the above bytecode (hex representation): |
| | | ```
// V8 Ignition bytecode (simplified)
LdaConstant [0]         // Load constant 'Hello world'
Star r0                 // Store in register r0
CallRuntime [console.log]  // Call console.log
Return                  // Return
``` | ```
48 8B 05 XX XX XX XX   // mov rax, [rip+offset]
48 89 C1              // mov rcx, rax
48 8D 15 XX XX XX XX  // lea rdx, [rip+offset]
E8 XX XX XX XX        // call <console.log>
C3                    // ret
``` |
| **Who processes it?** | Humans write it; compilers/interpreters process it | **Virtual Machine** (e.g., JVM, V8 engine, CPython VM) | **CPU (Central Processing Unit)** directly |
| **Portability** | Portable — same source runs anywhere the runtime exists | Portable — bytecode runs on any platform with a compatible VM | **Not portable** — tied to a specific CPU architecture (x86, ARM, etc.) |
| **Speed** | Slowest (must be parsed/compiled at runtime) | Faster than source code, slower than binary | **Fastest** — no translation step needed |
| **Generation Step** | Written by the developer | Generated by the **compiler/parser** (e.g., V8 parser) from source code | Generated by the **JIT compiler** or **AOT compiler** from bytecode |
| **Where does it live?** | `.js`, `.py`, `.java` files | `.class` (Java), `.pyc` (Python), or in-memory (V8) | `.exe`, `.out`, or in-memory after JIT compilation |
| **Real-world flow** (JavaScript) | `01-hello-world.js` | **V8 Parser** → **Ignition (Interpreter)** → Bytecode | **V8 TurboFan (JIT Compiler)** → Optimized Machine Code |

## Key Takeaway

> **Source code** is what you write. **Byte code** is an intermediate step the engine uses for efficiency. **Binary code** is what the CPU actually runs.

JavaScript (`01-hello-world.js`) starts as **source code**, gets parsed into **bytecode** by the V8 Ignition interpreter, and then hot code is **JIT-compiled** into optimized **binary machine code** by TurboFan for direct CPU execution.
